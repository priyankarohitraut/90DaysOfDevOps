Task 2: Create Your Profile README
Create a special repository with the same name as your GitHub username (e.g., github.com/yourname/yourname)
Add a README.md — this will appear on your profile page
Include the following in your profile README:
A short introduction — who you are, what you're learning
What you're currently working on (e.g., 90 Days of DevOps)
Skills/tools you know or are learning (Linux, Git, Python, Shell, etc.)
Links to your important repos
How to reach you (LinkedIn, Twitter, email — whatever you're comfortable sharing)
Keep it clean and simple — don't overload it with badges and widgets
Step 1: Create Profile Repository

Your repository name must be exactly the same as your GitHub username.

Example:

github.com/priyankarohitraut/priyankarohitraut

Steps:

Go to GitHub

Click New Repository

Repository name = your username

Tick Add README file

Click Create repository

Task 5: Clean Up

Delete or archive repos that are empty, abandoned, or irrelevant
Rename any repos with unclear names
Make sure you're not exposing any secrets (.env files, API keys, passwords) in any repo — check your commit history too
1️⃣ Delete or Archive Unnecessary Repositories

Check your repositories and remove ones that are:

❌ Empty
❌ Test repos (example: test123, practice1)
❌ Abandoned projects

Options

Delete repository

Open repository

Go to Settings

Scroll down

Click Delete this repository

OR

Archive repository (if you want to keep it but hide activity)

Go to Settings

Click Archive this repository

2️⃣ Rename Unclear Repository Names

Bad repo names look unprofessional.

Example
Bad Name	Good Name
test	linux-practice
newrepo	docker-practice
scripts	shell-scripts
notes	devops-notes

Use:

✔ lowercase
✔ hyphen (-) instead of spaces
✔ meaningful names

Example:

shell-scripts
docker-practice
linux-commands
devops-notes
3️⃣ Check for Secrets (Very Important)

Make sure your repositories do NOT contain sensitive files.

Never upload:

❌ .env files
❌ API keys
❌ passwords
❌ tokens
❌ SSH keys

Example dangerous file:

.env
API_KEY=123456abcdef
DB_PASSWORD=secret
4️⃣ Add .gitignore

To prevent secrets from being uploaded, create a .gitignore file.

Example:

.env
*.log
node_modules/
__pycache__/
*.pem
5️⃣ Check Commit History

Sometimes secrets remain in old commits.

Check commits:

Open repository

Click Commits

Review files that were uploaded

If secrets were pushed accidentally:

Remove them

Change the password/API key immediately
